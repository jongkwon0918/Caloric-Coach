# Caloric Coach (테트리스 팀)
> **AI 기반의 음식 일일 칼로리 계산 및 건강체크 앱**

## 🗓 프로젝트 개요
- **진행 기간:** 2024-11-14 ~ 2025-06-13 (졸업작품)
- **팀원:** 김태우, 박종권, 서반석, 유정현, 조영일 (테트리스 팀)
- **기획 배경:**
  최근 간편식, 외식, 배달음식 등의 이용이 일상화되면서 많은 사람들이 자신의 영양소 섭취 상태를 정확히 파악하지 못하고 특정 영양소를 과다 섭취하는 문제가 증가하고 있습니다.
  이에 본 프로젝트는 사용자가 섭취한 음식 사진을 찍어 올리면, AI가 이를 분석하여 칼로리와 영양소를 확인하고 남은 칼로리와 초과된 영양소 정보를 시각적으로 제공하여 건강한 식습관을 유도하는 헬스케어 앱을 개발하고자 하였습니다.

## 사전조사 및 필요성
기존의 헬스케어 앱(밀*그램, 다*어트신, Y*ZIO, 인*앗 등)들은 음식 검색을 통한 기록 기능이나 바코드 스캔, 성분표 인식 등의 기능을 제공하지만, **사용자가 섭취한 음식 사진을 직접 찍어 자동으로 분석하고 식단을 기록해 주는 기능**은 부족했습니다.
이를 해결하기 위해, 사용자의 신체정보를 바탕으로 권장 섭취 칼로리를 계산하고, **다중 인식, 자동 분석, 성분표 인식 기능을 포함하는 AI 기반의 직관적인 시각화 앱**이 필요하다고 판단했습니다.

---

## 기술 스택
- **Frontend (App):** Java (Android), XML, Android Studio, Firebase Authentication
- **Backend:** Python, Flask, REST API
- **Database:** Firebase Realtime Database, Firebase Storage
- **AI & Data Analysis:** YOLOv5 (음식 이미지 다중/단일 분석), Google Cloud Vision API (성분표 이미지의 영양정보 텍스트 추출), Ngrok
- **Co-working**: Git, GitHub

**데이터 플로우:**
앱에서 사진 촬영/업로드 -> Ngrok -> YOLOv5 서버 (음식 분석) / Vision API (성분표 분석) -> 결과값 JSON 반환 -> 앱 화면 출력 및 Firebase 저장

## 앱 기능 세부 내용

### 1. 로그인 / 회원가입
- **Firebase Authentication**을 이용한 인증 시스템 구현
- 이메일, 비밀번호, 성별, 나이, 키, 몸무게 등 기본 정보 입력
- 비인가 방식 접근 시 로컬 저장소 캐시를 활용하여 로그인 화면으로 이동

### 2. 프로필 및 신체정보 기반 칼로리 계산
- 입력된 신체정보를 바탕으로 **BMI 공식을 통해 권장 섭취 칼로리를 계산**
- 계산된 정보를 마이페이지에 시각적으로 표시하며, 데이터 변경 시 실시간 반영

### 3. 서버 통신 (Flask & Ngrok)
- Python 기반의 **Flask 프레임워크**로 서버 구축
- **Ngrok**을 활용하여 로컬 서버를 외부 환경(모바일 앱)에서 접근 가능하도록 구성

### 4. AI 이미지 분석 (다중 / 단일 음식)
- **YOLOv5** 모델을 활용하여 사용자가 업로드한 음식 사진 객체 분석 (다중/단일 인식 지원)
- **다중 음식 인식:** 여러 음식이 포함된 이미지를 분석하여 각각의 음식을 박스(Bounding Box) 처리 후 JSON 형태로 최대 15가지 음식 인식 결과를 반환.
- **단일 음식 인식:** 인식된 객체를 바탕으로 음식의 영양소 정보를 Firebase에서 조회하여 출력. 등록 시 식단 및 영양소 소비 현황에 자동 반영됨.
- 음식 분석 결과 수정 가능. 인식 실패 또는 오차 발생 시 직접 검색하여 추가할 수 있음.

### 5. 성분표 이미지 텍스트 추출
- **Google Cloud Vision API**를 활용하여 과자나 음료 등의 성분표 이미지를 인식
- 이미지 내 텍스트를 파싱하여 칼로리, 탄수화물, 단백질, 지방, 당류, 나트륨 등을 추출하고 결과를 JSON 형식으로 반환하여 데이터화 및 식단 기록에 활용.

### 6. 직접 검색 및 기록
- AI 인식이 어려운 음식이나 검색을 원하는 경우, 검색창을 통한 음식 직접 검색 및 등록 기능 지원.
- 검색 기록은 Firebase에 저장되어 '최근 검색한 음식'으로 노출됨.

### 7. 달력 기반 식단 조회
- 달력의 특정 날짜를 클릭하면, 해당 일자에 저장된 영양소 데이터를 Firebase에서 조회하여 막대그래프 형태로 시각화
- 날짜별 섭취 영양소를 직관적으로 비교/확인 가능

### 8. 메인 페이지 대시보드 (영양소 섭취 현황)
- 사용자의 영양소 섭취 현황을 칼로리, 탄수화물, 단백질, 지방별 그래픽 형태 및 진행 바(Progress bar)로 표시
- 남은 칼로리와 목표 섭취량 초과 여부를 한눈에 확인할 수 있도록 시각적 피드백 제공
- 초과된 일일 칼로리를 내부 알고리즘으로 계산하여 Firebase에 저장된 운동 소모 칼로리 데이터를 기준으로 운동 추

## 테스트 및 성능평가

### 테스트 내역
1. **딥러닝 서버 모듈:** 이미지 전송 및 분석 결과 수신 정상 작동, JSON 구조 일치화, 이미지 전처리 및 NMS 최적화를 통한 성능 개선
2. **사용자 입력 모듈:** 음식명 검색 정확도 향상, 단위 섭취량 조절 후 Firebase 연동, 최근 검색어 로컬(SharedPreferences) 저장 테스트
3. **성분표 처리 모듈:** API 호출 및 이미지 인식 정확도 검증, 추출된 텍스트의 분류 및 매핑 로직 안정화
4. **회원/정보 관리:** 회원가입/로그인 예외 처리, 프로필 수정 시 Firebase DB 업데이트 실시간 반영 확인

### 성능 평가 결과
- **성능:** 대부분의 기능은 2초 이내 응답하며, 이미지 분석은 최대 2초 이내 완료됨. 실시간 데이터베이스 저장 지연 없음.
- **편의성:** 검색 자동완성 팝업, 달력, 진행 상태바(Progress Bar) 등을 이용한 직관적인 UI/UX 설계.
- **신뢰성:** 입력 실패에 대한 적절한 예외 처리, 비정상적 조작에 대한 에러 메시지 출력 등 안정적인 동작 보장.
- **경제성:** 외부 유료 API 사용을 최소화하고 중저가형 모바일 기기에서도 원활하게 동작하도록 최적화.
